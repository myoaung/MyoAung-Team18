# Working Agreement - Team 18

## Communication
- Use clear and respectful communication
- Provide constructive feedback
- Ask questions when unclear

## Code Standards
- Follow consistent coding style
- Write meaningful commit messages
- Review code before merging

## Git Workflow
- Create feature branches for new work
- Submit pull requests for review
- Keep main branch stable

## Meetings
- Regular check-ins as needed
- Document decisions and action items

## Responsibilities
- All members contribute to code review
- Share knowledge and support team members
- Meet deadlines and communicate blockers
